## モチベーション

VSCode 界隈[^1]が AI コーディング IDE の最先端であることに異論はないが、AI 関連機能やエコシステムの充実度を除いた IDE の完成度では JetBrains IDE（以下 JetBrains）の方が個人的に優れていると思っている。
そこで、JetBrains で AI 機能を最大限に活かすにはどのような使い方をすべきか一度整理しておきたい。

[^1]: VSCode をベースにした Cursor や Windsurf なども含めて

前提として、JetBrains はエンジニアとしてのキャリアをスタートした時から使っているので愛着があるけど、他の IDE を使う方が生産性が上がると判断すれば容赦なく乗り換えたいと考えている。
実際、半年ほど前に Cursor に乗り換えたけど、使いにくかったのと Junie や Claude Code の登場とで JetBrains に回帰した経緯がある。

要は「彼を知り己を知れば百戦殆からず」[^2]ということわざがこの記事を書くモチベーションになっている。
AI 時代の敗者にならないように、まずは自分の使っているツールを整理して、その上で他のツールとの比較検証を行う地盤を作るのがこの記事の目標。

[^2]: 敵と味方の状況をよく理解していれば、何度戦っても負けることはないという意味のことわざ

## 使用しているツールの活用方法など

### Junie

Junie は JetBrains の AI コーディングエージェントである。
Claude Code に比べると、簡単なタスクであれば Junie の方が実行速度が速く、十分な性能を持っている。
なので、ユニットテストの作成やコンテキストを必要なリファクタリングであれば Junie を使うようにしている。

Junie には `CLAUDE.md` と同様の意味を持つ、`.junie/guidelines.md` というファイルがある。
このファイルに書かれた内容が AI に実行内容の文脈として渡される。

実質的に `CLAUDE.md` と同じ内容なので `.junie/guidelines.md` へのシンボリックリンクを作成したが、うまく読み込まれずにエラーが発生してしまう。
面倒だがコピペすることで対応しているが、AI エージェント間のメモリファイルを 1 つに統合できれば楽になる。

先述した通り、比較的簡単なタスクを任せているのに向いているが、以下の欠点があるので注意が必要（今後改善される可能性は高い）。

- LLM モデルを選択できない
- MCP が使えない
- 基本的にコマンド実行アクションの実施が必要（Brave モードは怖くて使えない）
  - Claude Code みたいにコマンドごとに許可できると良いのだが
- 稀にエラーで実行できない時がある（ライセンスの問題？）

他の AI コーディングエージェントの活用方法とも被るのだが、とにかく全ての作業を Junie にやらせてみる方法がまずはおすすめ。
Junie で対応できるのであれば Junie に任せて、それ以外のタスクは Claude Code に任せるのが現時点ではベストだと思っている。
Junie 一本で AI コーディングをしていくのはまだまだ難しい印象がある。
